# Variable matrix contract

Status: reference source of truth for place and corridor coverage. This file complements
`docs/data/variable-registry.md`, which names the full variable universe and source classes.

Purpose: make every place dossier comparable, deep, and honest. The matrix ensures Chania,
Heraklion, Valencia, Lisbon, and future places are evaluated against the same row set, while
preserving the correct granularity of each variable.

## Core rule

Every place is evaluated against the same canonical relocation variable matrix. A page may have
different coverage depth, but it must not have a different hidden standard.

National variables are correct when the rule is national. Local variables are required when the
reader's lived reality differs locally. Missing values must be explicit.

Every KEEP row in `docs/data/variable-registry.md` is part of the canonical source universe. A
city or region evidence bundle must instantiate every KEEP row. As of 2026-06-28, that means 254
rows in registry order. It may resolve a row as cited, inherited, proxy, unavailable, blocked, or
deferred, but it may not skip a row because it was inconvenient to source in the current pass.

The 254-row bundle is the evidence standard, not the Place object shape. Identity fields such as
slug, canonical coordinates, country, parent region, and route metadata live on the Place object.
The evidence bundle carries one row for each canonical variable and the source state for that row.

`out_of_slice` is not a completion state for a KEEP row that matters to relocation evaluation. It
is allowed only when a row is explicitly excluded from the current canonical matrix tier by a
registry decision or accepted ADR. Rows such as recorded crime, noise pollution, religiosity,
social trust, company formation, childcare cost, and public health coverage are KEEP rows. If they
are not yet sourced, they must be tracked as `source_search_required` until source discovery either
produces a CitedValue or closes the row with a real source-gap reason.

The decision-useful city-signal rows in `docs/data/variable-registry.md` section 9 are part of the
city template for every standard town page. They are the rows most likely to answer client and
AI-crawler questions about whether a place works year-round, for families, for remote work, for
retirees, and for city-to-city comparison. More rows are useful only when they are sourceable,
comparable, and answer a real query.

## Compiler operation

The city matrix is compiled, not researched row by row. A matrix run generated by
`scripts/plan-city-matrix.ts` assigns every row a `compilerLane`, `sourceFamily`, and `batchScope`.
Those fields decide how the row should be filled.

| Field | Meaning |
|---|---|
| `compilerLane` | The operating lane, such as inherited national, geospatial adapter, AI crawl, licensed/manual, derived formula, runtime, or final gap |
| `sourceFamily` | The batch unit, such as Eurostat country data, OSM/OSRM routing, Open-Meteo environment data, tax/residency manual sources, or property terms-limited sources |
| `batchScope` | The run scope: country, region, cluster, town, runtime, or manual batch |

The run must follow `summary.laneRunOrder` and `laneBatches`. This keeps future towns from
becoming 254 bespoke research tasks. Run country rows once, region rows once, source adapters
across the whole cluster, then AI crawl only the rows that genuinely need official text search.

Implemented source adapters must also be declared in
`docs/data/city-matrix-adapter-registry.json`. The registry lists adapter ids, row keys,
compiler lane, source family, batch scope, command, and proxy caveat. A row-producing adapter
without a registry entry is incomplete, because future city pulls need to know which batch can
close which row.

Use `scripts/report-city-bundle-readiness.ts` after each batch to read the evidence bundles and
report rows, cited rows, completed gaps, source-search rows, high-liability freshness blockers,
local distinctness, and Page Factory status.

For draft bundles that are not registered in the public bundle index yet, pass explicit ids:

```bash
./node_modules/.bin/tsx scripts/report-city-bundle-readiness.ts --bundles pt-lisbon,pt-lisbon-ericeira
```

### Bootstrap operation for new cities

For a city that does not yet have an evidence bundle, run the compiler and then seed the bundle
before doing source work. The seed pass is a work-queue generator, not a publication gate.

Example:

```bash
pnpm data:city-matrix "Lisbon" --country Portugal --region Lisbon --id pt-lisbon --run --write
pnpm data:seed-city-bundle "Lisbon" --country Portugal --region Lisbon --country-place pt --id pt-lisbon --lat 38.7077507 --lon -9.1365919 --coordinate-source-url https://www.openstreetmap.org/relation/5400890 --coordinate-source-name "OpenStreetMap relation 5400890, Lisboa administrative boundary" --write
```

The seeder must instantiate all 254 registry rows, inherit only existing country CitedValues with
national granularity, close known relational or source-policy rows as completed gaps, and leave
the remaining unresolved rows as `source_search_required`. This saves time because every new city
starts with the same compiled source-family board, but it does not make the city page-ready.

After seeding, the default automation goal is source-family closure, not manual row hunting. Run
country-safe adapters across the whole target cluster first: UV index, pollen severity, terrain,
surface water, forest or tree cover, drought frequency, light pollution, seasonal-service dropoff,
and emergency-vet proxies. Then run country-specific official adapters, such as GTFS/transit,
airport route, rail/ferry, health registry, tourism, beach, water, and municipal-service sources.
Only after those batches should AI crawl remaining official-source rows.

The publish gate is separate from full data fill. `summary.publishGate.requiresAllRowsFilled` must
remain false. Page Factory needs all 254 rows attempted, no `source_search_required`, fresh
high-liability rows, honest inherited/proxy labels, completed source-gap reasons, and enough local
distinctness. It does not need every row to have a CitedValue.

## Vocabulary

| Term | Meaning |
|---|---|
| Variable | A stable fact type, such as `top_pit_rate`, `dnv_income_floor`, `airport_type`, or `cafe_density` |
| Intended granularity | The granularity the variable naturally belongs to |
| Observed granularity | The granularity of the source actually used |
| Coverage status | Whether the value is available, inherited, proxy, unavailable, or blocked |
| Evidence bundle | The resolved matrix values and source gaps for one candidate |
| Dossier | A public page or internal page draft produced from an evidence bundle |

## Granularity classes

### National by nature

These values should normally be shared by all places in the same country:

- tax rates and special tax regimes;
- visa and residency routes;
- Schengen membership;
- citizenship timelines;
- national healthcare eligibility;
- national education indicators;
- national safety and rights indices;
- national English-proficiency or social-trust indices;
- national price-level and inflation metrics.

For example, Chania and Heraklion share Greece's national tax rules. This is correct and should
be shown as country-level data.

### Regional by nature

These values often sit above a town but below a country:

- hospital catchment and regional health capacity where sourced;
- region-level property indices;
- regional airports and transport catchment;
- NUTS regional economic or demographic data;
- regional climate or hazard bands where the source is regional;
- school district or municipality-level services when the city boundary is not the right unit.

### Local by nature

These values should differ between places when sources allow:

- airport drive time and airport type;
- ferry and rail access;
- local rental and asking-price proxies;
- hospital, pharmacy, dentist, gym, cafe, school, childcare, and service density;
- walkability and amenity density;
- beaches, bathing-water sites, hiking access, green space, tree cover, wildfire/flood risk;
- broadband or mobile speed where tile data exists;
- tourism pressure and seasonality proxies;
- local climate normals and sea-temperature access where point-sampled.

### Relational by nature

These values depend on the reader's input:

- treaty with the reader's home country;
- totalization agreement with the reader's home country;
- flight time to the reader's home hub;
- timezone overlap with the reader's clients;
- citizenship by descent for the reader's ancestry;
- pet-import route from the reader's origin country.

Relational values are not stored as fixed place facts. They are computed from cited base facts
and rendered with the same fence.

## Coverage status enum

Every evaluated matrix row should resolve to one status.

| Status | Meaning | Render behavior |
|---|---|---|
| `local` | Source is place-level or point-sampled to the place | Render as local |
| `regional` | Source covers region or catchment | Render with regional label |
| `national` | Source covers country and variable is national by nature | Render as national |
| `inherited_national` | Source covers country but variable is displayed on a lower-place page | Render "national figure shown for this place" |
| `inherited_regional` | Source covers region but displayed on town page | Render "regional figure shown" |
| `proxy` | Source is an explicit proxy, such as OSM amenity density | Render proxy label |
| `relational` | Computed from reader input and cited base facts | Render with input context |
| `unavailable` | No reliable public source found | Render "no reliable source found yet" |
| `blocked` | Source exists but cannot be used legally or technically | Render source-gap note internally, do not publish as fact |
| `deferred` | Useful but out of current slice | Internal queue only |

`blocked` plus `source_search_required` means the row is required, but the Data Desk has not yet
completed source discovery. It is not the same as `unavailable`. It is not a zero. It blocks
`data_bundle_ready`.

## Matrix categories

These categories are the operating layer over the registry's detailed variables.

### Identity and geography

Minimum fields:

- stable id;
- slug;
- country;
- parent region;
- coordinates;
- population where sourced;
- administrative unit;
- coastline, island, mountain, or inland tags where sourced.

### Money and cost

Minimum fields:

- price level index;
- inflation;
- electricity or utility price where available;
- rent or asking-price proxy;
- housing burden or property index;
- grocery and restaurant proxies when official data is unavailable;
- mortgage or transfer-cost fields for ownership corridors.

### Tax and residency

Minimum fields:

- headline personal income tax rate;
- special inbound or pension regime;
- wealth, inheritance, property, dividend, and capital-gains variables when relevant;
- DNV, passive-income, investor, or startup route variables where available;
- PR and citizenship timelines;
- Schengen membership;
- treaty and totalization hooks for relational tools.

### Climate and seasonality

Minimum fields:

- Koppen class;
- monthly high and low temperatures;
- warmest-month high;
- coldest-month low;
- precipitation;
- winter daylight;
- clear-sky proxy;
- sea temperature and swimming season for coastal places;
- wildfire, flood, seismic, and heat-risk variables where sourced.

### Travel, connectivity and work

Minimum fields:

- nearest airport;
- airport distance or drive time;
- airport type;
- ferry, rail, or major road access where relevant;
- broadband and mobile speed;
- timezone;
- transit, walkability, and EV/bike-share proxies where relevant;
- company-formation or business friction at country level.

### Health, family and schooling

Minimum fields:

- physicians per 1,000;
- hospital beds;
- hospital and pharmacy local density;
- paediatric or elder-care source where available;
- vaccination and healthcare coverage country values;
- childcare cost and enrolment;
- international school, Montessori, Waldorf, or bilingual-provision signals where sourced;
- family-leave and education indicators.

### Safety, rights and governance

Minimum fields:

- GPI or safety index;
- homicide or recorded crime where sourced;
- corruption, rule of law, government effectiveness;
- women safety and LGBTQ+ rights/acceptance variables;
- press freedom;
- political stability;
- local crime only if a reliable local source exists.

### Nature and environment

Minimum fields:

- PM2.5 or air-quality proxy;
- distance to coast;
- elevation;
- bathing-water quality for coastal places;
- green space, forest cover, tree canopy;
- protected areas and national-park access;
- hiking, climbing, surf, or outdoor proxies where sourced;
- light pollution and noise where sourced.

### Culture, community and services

Minimum fields:

- English proficiency;
- foreign-born share;
- cafe, restaurant, gym, library, museum, theatre, pool, and service-density proxies;
- builders, plumbers, electricians, vets where OSM coverage is strong enough;
- parkrun or club proxies where sourced;
- smoking, alcohol, social trust, religiosity, and cannabis status at country level where relevant.

### Decision-useful city signals

Minimum fields for standard and flagship town pages:

- emergency-hospital access and drive-time rows;
- rental supply, long-let supply, furnished-rental share, and asking-rent rows where sources allow;
- airport winter/summer route rows, ferry winter/summer route rows, bus-frequency rows, and car-dependency proxy;
- playground, nursery, school, family-doctor, stroller-hilliness, and family-amenity proxy rows;
- tourist beds, overnights, seasonality, short-stay pressure, cruise pressure, and seasonal-service-dropoff rows;
- residence office, tax office, citizen-service centre, land-registry office, and municipal digital-service rows;
- water stress, drought, water restriction, power reliability, heatwave, cooling-degree-day, and wildfire-egress rows;
- coworking, language-school, international-school, pet, slope, sidewalk, wheelchair, accessible-beach, and public-toilet rows.

These rows should feed answer blocks such as "best Crete city for families", "year-round living in
Rethymno", "Chania vs Heraklion hospital access", "Agios Nikolaos without a car", and "which Crete
city is most seasonal." The page may summarize them, but the evidence bundle must retain each
underlying row and source state.

## Coverage tiers

| Tier | Use | Minimum coverage |
|---|---|---|
| `pilot` | Internal proof page, explicitly labelled in queue | 254 rows attempted, 40 resolved or completed-gap rows, 20 populated, 6 local/regional. Pilot does not count as Page Factory handoff unless the queue says so. |
| `standard` | Normal publishable dossier | 254 rows attempted, no `source_search_required` rows, 35 populated, 12 local/regional |
| `flagship` | Major SEO/AEO place page | 254 rows attempted, no `source_search_required` rows, 55 populated, 20 local/regional, priority city-signal rows cited or completed-gap closed |
| `comparison_ready` | Suitable for cross-place comparison | Same variable keys populated or explicitly missing across all compared places |

The `standard` tier is the default for new place pages. Thresholds such as 35 populated rows are
minimum publish-depth signals, not permission to omit the rest of the 254-row template.

## Variable row shape

Queue and bundle files should use this shape when a value is evaluated.

```json
{
  "key": "airport_drive_minutes",
  "label": "Drive time to nearest commercial airport",
  "intendedGranularity": "town",
  "observedGranularity": "town",
  "coverageStatus": "local",
  "sourceClass": "manual",
  "value": {
    "value": 22,
    "sourceName": "Manual route check",
    "sourceUrl": "https://example.com",
    "verifiedDate": "2026-06-28",
    "confidence": "low",
    "granularity": "town",
    "category": "connectivity"
  },
  "unit": "minutes",
  "sourceGapReason": null,
  "notes": "Manual route checks should be refreshed before publication."
}
```

The example source URL is illustrative only. Do not commit example values into production data.

## Publish calculation

A place dossier passes the matrix gate when:

1. Every category has at least one evaluated row.
2. Tax, visa, and residency rows are fully cited and fresh.
3. Local-by-nature rows are not silently replaced with national values.
4. National-by-nature rows are not penalised for being national.
5. Every canonical row required for the place tier is present as a row.
6. Important missing rows are visible in the source-gap list.
7. No row is left as `source_search_required`.
8. Comparison pages use the same keys across every compared place, or mark missing keys.
9. Standard and flagship town pages evaluate the full decision-useful city-signal layer, with no
   silent omissions for sourceable transport, healthcare, housing, family, tourism,
   infrastructure, pet, or accessibility signals.

## AI-citation calculation

A city page is more citable when it has clear answers backed by rows, not when it has a larger
uncurated table. For each city bundle, tag the rows that support these answer shapes:

- year-round practicality;
- family practicality;
- healthcare access;
- airport and ferry access;
- rental and housing reality;
- tourism pressure;
- car dependency;
- retirement practicality;
- remote-work practicality;
- accessibility and hilliness;
- pet practicality.

Do not invent a composite score until every component row is cited or source-gap closed. Composite
rows are allowed only as derived facts with their own source trail back to the component rows.

## Source-gap reasons

Use consistent reasons so automation can aggregate them:

- `no_public_source_found`
- `source_exists_but_paywalled`
- `source_terms_block_reuse`
- `source_bot_blocked_manual_needed`
- `local_source_too_sparse`
- `official_source_only_national`
- `requires_geospatial_build`
- `requires_manual_maps_check`
- `source_search_required`
- `out_of_slice`

Use `source_search_required` only as a temporary blocker. After source discovery, convert it to a
cited value or to the real completed gap reason, such as `no_public_source_found`,
`source_terms_block_reuse`, `local_source_too_sparse`, or `official_source_only_national`.

Paywalled or terms-limited rows may be accepted as publication-visible gaps only after the bundle
names the approved source family, records why scraping or substitution is barred, and keeps the row
as a visible gap unless licensed or terms-cleared access is added.

## How this connects to the code

Current code already supports:

- `Place.variables` as a map of CitedValues;
- place evidence bundles with 254 canonical rows per Crete bundle;
- profile presets;
- slice-1 variable catalogue;
- cited value collection and freshness checks.

The next implementation step is to keep the 254-row evidence-bundle contract and the
`docs/data/variable-registry.md` registry in sync. `packages/data/src/variables/catalog.ts` can
still expand in batches, but Page Factory readiness is judged from the evidence bundle and queue
state, not from a smaller frontend variable catalogue.

## Quality bar

A strong dossier can say:

- "This tax value is national because Greece has national tax law."
- "This hospital value is regional because the source is regional."
- "This cafe-density value is local and sourced from OpenStreetMap."
- "This school value is unavailable because no reliable public source was found."

That honesty is the product. It is what lets many pages share one matrix without becoming thin.
